# 반응형 디자인 개선 사항

## 개요

스마트폰과 태블릿 화면에서 UI가 깨지지 않도록 반응형 디자인을 구현했습니다.
웹 개발의 반응형 디자인 원칙을 Flutter 앱에 적용했습니다.

## 주요 변경 사항

### 1. Settings Screen (설정 화면)

#### 헤더 및 검색 바

- **LayoutBuilder** 사용하여 화면 너비 감지
- 좁은 화면 (< 600px):
  - 패딩: 16px → 8px
  - 검색 힌트 텍스트: "Search settings..." → "Search..."
  - 아이콘 버튼 패딩 감소
  - 입력 필드 패딩 감소

#### 설정 콘텐츠

- 좁은 화면에서 ListView 패딩 감소 (16px → 8px)

#### 드롭다운 설정

- **반응형 레이아웃 전환**:
  - 넓은 화면: 가로 레이아웃 (제목 - 드롭다운)
  - 좁은 화면: 세로 레이아웃 (제목 위, 드롭다운 아래)
- 좁은 화면에서 드롭다운이 전체 너비 사용
- 오버플로우 문제 해결

#### 컬러 스킴 설정

- 좁은 화면에서 칩 간격 감소 (8px → 4px)
- 좁은 화면에서 폰트 크기 감소 (14px → 12px)
- 칩이 더 잘 감싸지도록 개선

### 2. Session List Screen (세션 목록 화면)

#### 상단 탭 바

- **LayoutBuilder** 사용하여 반응형 구현
- 좁은 화면 (< 600px):
  - 탭 라벨 패딩: 16px → 8px
  - 탭 너비: 120px → 100px
  - 아이콘 크기: 20px → 18px
  - 세션 탭 텍스트 크기: 14px → 12px
  - 닫기 버튼 크기 감소
  - 버튼 패딩 최적화

#### 검색 바

- 좁은 화면에서 패딩 감소 (16px → 8px)
- 검색 힌트 텍스트: "Search sessions or tags..." → "Search..."
- 입력 필드 패딩 조정

#### 세션 카드

- **완전한 반응형 구현**:
  - 카드 마진: 16px → 8px (가로), 8px → 4px (세로)
  - ListTile contentPadding 추가
  - CircleAvatar 크기 조정 (radius: 20 → 18)
  - 아이콘 크기 조정 (24 → 18)
  - 제목 폰트 크기: 16px → 14px
  - 부제목 폰트 크기: 14px → 12px
  - 태그 폰트 크기: 12px → 10px
  - 액션 버튼 패딩 감소 (8px → 4px)
  - 텍스트 오버플로우 처리 (ellipsis)

## 기술적 구현

### 반응형 브레이크포인트

```dart
final isNarrow = constraints.maxWidth < 600;
```

- 600px를 기준으로 좁은 화면과 넓은 화면 구분
- 스마트폰: < 600px (좁은 화면)
- 태블릿/데스크톱: >= 600px (넓은 화면)

### 사용된 Flutter 위젯

- **LayoutBuilder**: 부모 위젯의 제약 조건을 기반으로 반응형 레이아웃 구축
- **MediaQuery**: 화면 크기 정보 접근 (간접적으로 사용)
- **Flexible/Expanded**: 유연한 레이아웃 구성

### 적용된 반응형 패턴

1. **적응형 패딩**: 화면 크기에 따라 패딩 조정
2. **적응형 타이포그래피**: 작은 화면에서 폰트 크기 감소
3. **적응형 아이콘**: 작은 화면에서 아이콘 크기 감소
4. **레이아웃 전환**: 가로 ↔ 세로 레이아웃 전환 (드롭다운)
5. **텍스트 오버플로우 처리**: ellipsis 사용

## 테스트 권장 사항

다음 화면 크기에서 테스트하는 것을 권장합니다:

- 📱 스마트폰: 360px - 480px (좁은 화면)
- 📱 큰 스마트폰: 480px - 600px (좁은 화면)
- 📱 태블릿: 600px - 900px (넓은 화면)
- 💻 데스크톱: > 900px (넓은 화면)
